# Система подбора водителей

## Описание
Реализация алгоритмов поиска ближайших водителей для заказов. Три различных подхода к решению задачи с различной временной сложностью.

## Архитектура
- `DriverMatching.Core` — базовые модели и интерфейсы
- `DriverMatching.Algorithms` — реализация алгоритмов
- `DriverMatching.Tests` — модульные тесты
- `DriverMatching.Benchmarks` — сравнение производительности

## Алгоритмы

### 1. Linear Search
- Временная сложность: O(N log N)
- Подход: вычисление расстояния до всех водителей и сортировка

### 2. Priority Queue
- Временная сложность: O(N log K), где K = 5
- Подход: поддержание очереди из 5 ближайших водителей

### 3. Grid-Based
- Временная сложность: O(N) в среднем случае
- Подход: разбиение карты на сетку, поиск в ближайших секторах

## Результаты бенчмарков (BenchmarkDotNet)

![Результаты бенчмарков](benchmark.png)

| Алгоритм             | 100 водителей | 1000 водителей | Память (100) |
|----------------------|---------------|----------------|--------------|
| Linear Search        | 8.991 μs      | 132.422 μs     | -            |
| Priority Queue       | 6.867 μs      | 61.022 μs      | 1888 B       |
| Grid-Based           | 9.143 μs      | 129.108 μs     | 5560 B       |

### Выводы:
- **Priority Queue** показал лучшее время при обоих размерах данных
- **Grid-Based** потребляет больше памяти, но не дал ожидаемого выигрыша по времени
- **Linear Search** — самый простой, но медленный при большом количестве водителей
